1. No, you wouldn't use a unit test to test the "message" feature of a message application since this is a larger scale feature that interacts with many elements of the program. A unit test would fail to test how the individual components of the application (such as type, send, recieve, etc) interact with each other on a application/feature level.
2. Yes, you would use a unit test to test the "max message length" feature of a messaging application since it is small scale feature that limits one aspect of the applitcation. It can executed and tested quickly to debug without taking too much time and effort.
